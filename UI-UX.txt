1. PROJECT OVERVIEW
================================================================================

SYSTEM NAME: YÃ¼kleme YÃ¶netim Sistemi (Loading Management System)
LANGUAGE: Turkish (tr)
DIRECTION: Left-to-Right (LTR)
PURPOSE: A web-based application for managing truck loading operations at a factory/warehouse facility.

CORE FUNCTIONALITY:
- Track and document truck loading operations
- Record vehicle information, driver details, and loaded goods
- Capture photographic evidence of goods before loading
- Allow production managers to review and mark records as "recorded" in the system
- Provide real-time statistics and searchable records

PRIMARY USE CASE:
Factory workers document each truck loading with detailed information including team members, vehicle plates, weights, destinations, products loaded, timestamps, and photos. Production managers can then review all submissions and mark them as "recorded" once processed in the main system.

2. USER ROLES & AUTHENTICATION
================================================================================

ROLE 1: LOADER (YÃ¼kleme Sorumlusu)
----------------------------------
- Username: loader
- Access Level: Can create new loading records
- Dashboard: Loader Dashboard with data entry form
- Permissions:
  * Submit new loading records
  * Upload photos and videos
  * View own submissions (optional)
  * Cannot view other loaders' submissions
  * Cannot mark records as "recorded"

ROLE 2: MANAGER (Ãœretim MÃ¼dÃ¼rÃ¼)
-------------------------------
- Username: manager  
- Access Level: Full read access, can update record status
- Dashboard: Manager Dashboard with records table
- Permissions:
  * View all loading records
  * View detailed information for each record
  * Mark records as "Recorded" (Kaydedildi)
  * Cancel/undo "Recorded" status
  * Cannot create or edit loading records

AUTHENTICATION FLOW:
1. User navigates to login page
2. User enters username and password
3. System validates credentials
4. On success: Redirect to appropriate dashboard based on role
5. On failure: Display error message with shake animation
6. Session persists until explicit logout or timeout
